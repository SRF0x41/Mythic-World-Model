# Mythic World Model

The Mythic World Model is a research system designed to construct an evolving representation of the contemporary zeitgeist.

At its core, however, the Mythic World Model is **not simply an AI pipeline—it is a database of knowledge**. Every object generated by the system is persisted, versioned, and linked to every other relevant object. Rather than treating analysis as disposable output, the system treats reasoning itself as data.

The result is a living knowledge base that continuously grows over time. New information does not replace old information; it enriches and reshapes the network of evidence, concepts, and explanations that make up the model.

The purpose of the project is not merely to answer isolated questions, but to accumulate understanding. It searches for recurring patterns, stable concepts, causal relationships, and ultimately coherent explanations of cultural change.

Nothing is discarded. Every observation remains available as evidence for future analysis.

The system transforms raw information into increasingly abstract layers of understanding.

---

# Philosophy

Most modern AI systems are designed around conversations. They ingest information, produce an answer, and then forget the reasoning process that created it.

The Mythic World Model takes the opposite approach.

Every intermediate step becomes a permanent object within the database. Sources become Evidence. Evidence becomes Signals. Signals become Concepts. Concepts become Hypotheses. Hypotheses become Models. Models generate Predictions. Collections of Models become World Models.

Because every layer is stored, the entire reasoning chain remains inspectable and reusable. Any conclusion can be traced back to the original evidence that produced it.

The database is therefore not simply storing documents—it is storing **knowledge structures**.

---

# Ontology

The conceptual backbone of the Mythic World Model is its ontology.

An ontology is a formal representation of the important things that exist within a domain and the relationships between them. Rather than viewing information as isolated documents, an ontology represents knowledge as interconnected objects.

Concepts form the ontology of the system.

Each Concept represents a stable entity within the world model:

- Trust
- Religion
- Community
- Identity
- Institutions
- Technology
- Ritual
- Individualism
- Economic Security

These are not merely keywords. They are persistent objects with relationships to other concepts.

For example:

```
Religion
    ├── influences → Community
    ├── opposes → Secularization
    ├── related to → Ritual
    └── contributes to → Shared Identity
```

As the ontology grows, it becomes a semantic graph that allows the system to reason beyond simple keyword matching.

Instead of asking:

> "Which articles mention religion?"

the system can ask:

> "Which concepts connected to Religion have experienced significant increases in supporting evidence over the past five years?"

or

> "What pathways connect Institutional Distrust to Declining Birth Rates?"

The ontology provides the conceptual map that every higher layer of reasoning depends upon.

---

# Why an Ontology?

Without an ontology, the system remains a large collection of documents.

With an ontology, the database becomes a structured model of reality.

The ontology allows the Mythic World Model to:

- connect related ideas across unrelated sources
- accumulate evidence around enduring concepts rather than transient words
- discover indirect relationships through graph traversal
- support multiple competing explanations using the same underlying concepts
- continuously refine conceptual definitions as new evidence arrives

The ontology transforms the database from an archive into a knowledge graph capable of supporting genuine world modeling.

Concepts become the "vocabulary" from which hypotheses and models are constructed.

# System Architecture & Tech Stack

The Mythic World Model is intentionally built on a **minimal, transparent, and fully local-first tech stack**.

The entire system is implemented using:

- **Python**
- **SQLite**

No distributed systems, no external databases, and no heavyweight infrastructure layers are required.

This design choice is deliberate.

Python serves as the reasoning and orchestration layer: ingesting sources, extracting evidence, generating signals, constructing ontological relationships, and managing model evolution.

SQLite serves as the persistent knowledge substrate: storing every object in the system as structured relational data while preserving full historical traceability.

Together, they form a single unified system:

- Python defines how knowledge is interpreted, transformed, and connected.
- SQLite guarantees permanence, consistency, and full historical lineage.

This minimal stack enforces clarity. Every transformation is explicit. Every object is inspectable. Every relationship is queryable.

There is no hidden state and no external dependency required for the system to function.

# Conceptual Workflow

## Layer 1 — Sources

Everything begins as a **Source**.

A Source is an immutable record of raw information such as:

- News articles
- Academic papers
- Reddit discussions
- Government publications
- Books
- Interviews
- Podcasts
- Surveys
- Transcripts

Sources are never modified or deleted.

Each Source contains:

- Unique ID
- URL or origin
- Author
- Publication date
- Source type
- Credibility score
- AI-generated summary
- Raw text

Sources are organized into independent databases based on origin (Articles, Reddit, Government, Papers, etc.) while sharing a common schema.

**Question answered**

> What information do we possess?

---

## Layer 2 — Evidence

Sources are transformed into **Evidence**.

Evidence represents a single extractable claim supported by a Source.

One article may generate many independent pieces of evidence.

Each Evidence object contains:

- Unique ID
- Claim
- Supporting quotation or extracted passage
- Source reference
- Confidence score
- Related Evidence

Evidence is the atomic unit of reasoning.

**Question answered**

> What does this source actually support?

---

## Layer 3 — Signals

Evidence accumulates into **Signals**.

Signals answer one fundamental question:

> What keeps showing up?

Signals identify recurring observations across many independent sources.

Examples include:

- Rising institutional distrust
- Declining religious participation
- AI anxiety
- Increasing loneliness
- Political fragmentation
- Housing insecurity

Each Signal contains:

- Signal claim
- Supporting Evidence
- Confidence score
- Frequency
- Time distribution

Signals identify patterns but intentionally avoid explaining them.

**Question answered**

> What patterns consistently emerge?

---

## Layer 4 — Concepts

Signals are distilled into **Concepts**.

Concepts are stable ontological objects that change slowly over time.

Rather than representing observations, Concepts represent the enduring structures that exist beneath those observations.

Each Concept contains:

- Name
- Description
- Confidence score
- Parent Concepts
- Child Concepts
- Semantic relationships
- Related Signals
- Related Evidence

Concepts collectively form the ontology graph that underpins the entire system.

They are intended to be reusable across thousands of different analyses.

**Question answered**

> What enduring things are we observing?

---

## Layer 5 — Hypotheses

Hypotheses introduce causal reasoning.

They connect Concepts together through directional relationships.

Examples:

- Declining Religion increases Search for Secular Rituals.
- Institutional Distrust reduces Civic Participation.
- Social Media increases Identity Performance.
- Economic Insecurity amplifies Political Polarization.

Each Hypothesis contains:

- Claim
- Source Concepts
- Target Concepts
- Supporting Signals
- Supporting Evidence
- Confidence score
- Competing Hypotheses

Multiple hypotheses may coexist simultaneously.

The system allows competing explanations until accumulating evidence increases confidence in one over another.

**Question answered**

> How do these concepts influence one another?

---

## Layer 6 — Models

Models are networks of interconnected Hypotheses.

Rather than explaining isolated relationships, Models explain systems.

Example:

```
Economic Insecurity
        ↓
Institutional Distrust
        ↓
Loss of Shared Identity
        ↓
Search for Meaning
        ↓
Rise of Alternative Communities
```

Each Model contains:

- Network of Hypotheses
- Related Concepts
- Supporting Signals
- Supporting Evidence
- Confidence score
- Predictions
- Competing Models

Models compete based upon explanatory power rather than popularity.

Confidence increases as additional evidence supports the same network of relationships.

**Question answered**

> What system best explains the observed patterns?

---

## Layer 7 — Predictions

Predictions extend Models into the future.

Predictions are falsifiable expectations generated from a Model.

Examples:

- Interest in secular rituals will continue increasing.
- AI companionship will become socially normalized.
- Civic organizations will continue declining.
- Identity-based communities will become increasingly important.

Each Prediction contains:

- Prediction statement
- Parent Model
- Confidence score
- Expected timeframe
- Validation status

Predictions provide a mechanism for continuously evaluating the quality of the models.

**Question answered**

> If this model is correct, what should happen next?

---

## Layer 8 — World Models

World Models represent the highest level of abstraction.

A World Model synthesizes the strongest Models into a coherent explanation of the current historical moment.

Rather than presenting isolated trends, the World Model attempts to answer:

- What kind of era are we living through?
- Why are these cultural shifts occurring?
- Which underlying forces appear to be driving them?
- How do seemingly unrelated events connect?

Each World Model contains:

- Supporting Models
- Supporting Hypotheses
- Concepts
- Signals
- Evidence
- Sources
- Confidence score
- Narrative synthesis

A World Model is never considered complete.

As new information enters the system, every layer—from Sources through Models—may evolve, producing a continuously improving understanding of the world.

# Database Architecture

The Mythic World Model is fundamentally an object graph stored inside a database.

Every object maintains explicit references to the objects that produced it and the objects it supports.

For example:

```
Source
    ↓
Evidence
    ↓
Signal
    ↓
Concept
    ↓
Hypothesis
    ↓
Model
    ↓
Prediction
    ↓
World Model
```

Because every layer is persisted, the system gains several important properties:

- Complete provenance from every conclusion back to the original source.
- Incremental growth rather than repeated analysis.
- Reusable reasoning objects.
- Graph traversal across concepts and hypotheses.
- Multiple competing explanations without data duplication.
- Continuous refinement as new information arrives.

The database therefore becomes far more than document storage. It becomes an evolving representation of knowledge itself.

---

# Guiding Principle

The Mythic World Model is not designed to produce opinions.

It is designed to construct **testable, evidence-backed explanations**.

Every conclusion must be traceable through Models, Hypotheses, Concepts, Signals, Evidence, and ultimately the original Sources.

Nothing exists without provenance.

The long-term vision is to build a living cognitive map of the modern world—a continuously evolving knowledge graph capable of explaining how culture changes, why those changes occur, and what they may imply for the future.
